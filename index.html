Description

  • Dies ist die Ontologie für ARCHE. In den Definitionen der Eigenschaften, meint "Ressource A" das Subject und "Ressource B" das Objekt. When Definitionen aus anderen Schemata (z. B. DataCite) wiederverwendet werden, wird die Referenz in Klammern angegeben: (DataCite: ContactPerson). When die ursprüngliche Definition leicht angepasst wurde, ist die Referenz mit "cf." markiert.

  • This is the ontology for ARCHE. In definitions of properties, "resource A" refers to the subject resource of the property and "resource B" to the object resource. When reusing definitions from other vocabularies (esp. DataCite), the reference is given in brackets: (DataCite: ContactPerson). When the source definition was changed slightly the reference is marked with "cf."

Description

  • Entity able to act, in the context of this schema especially be involved in the making, processing, distributing, publishing etc. of a Resource, Collection, Concept or Project. Note that in contrast to foaf:Agent acdh:Agent can only be a Person or an Organisation (not a software or physical artifact). The different roles an Agent can assume with respect to a Project, Collection, Resource or Concept are modeled as separate object properties (sub-properties of acdh:contributedTo).

  • Handlungsfähige Entität, die im Rahmen dieses Schemas insbesondere an der Erstellung, Verarbeitung, Verteilung, Veröffentlichung usw. einer Ressource, Sammlung, eines Konzepts oder Projekts beteiligt ist. Beachten Sie, dass im Gegensatz zu foaf:Agent acdh:Agent nur eine Person oder eine Organisation sein kann (nicht eine Software oder ein physisches Artefakt). Die verschiedenen Rollen, die ein Agent in Bezug auf ein Projekt, eine Sammlung, eine Ressource oder ein Konzept übernehmen kann, werden als separate Objekteigenschaften (Unter-Eigenschaften von acdh:contributedTo) modelliert.

Dissemination Service Matching Rule c

IRI https://vocabs.acdh.oeaw.ac.at/schema#DisseminationServiceMatchRule
Description

  • Beschreibt eine Regel, die verwendet wird einen Dissemination-Service mit einer Repositioriumsressource zu verknüpfen. Eine Regel besteht aus einem {metadata property, value}-Paar, welches in den Metadaten der Ressource im Repositiorium vorhanden sein sollte, damit die Regel greifen kann. Eine Regel kann optional oder erforderlich sein. Ein Dissemination-Service wird auf eine Ressource angewendet, wenn alle erforderlichen Regeln und zumindest eine optionale Regel erfüllt werden.

  • Describes a rule used to match a dissemination service to repository resources. A rule consists of a {metadata property, value} pair which should be present in the repository resource's metadata for a rule to be matched. A rule can be optional or required. A resource matches a dissemination service if all required matched rules and at least one optional matching rule is matched.
